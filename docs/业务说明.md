# 软装物料出入库管理系统 - 业务说明文档

## 1. 系统概述

本系统旨在提供一套完整的软装物料管理方案，实现从采购计划到最终库存汇总的数字化追踪。系统通过建立统一的物料编号体系，将采购、基础资料、入出库流水及库存状态有机结合，确保物料流转的可追溯性和数据的实时性。

## 2. 核心业务流程

系统的运作遵循以下逻辑链路：
**定义采购计划 $\rightarrow$ 构建物料主库 $\rightarrow$ 记录入库流水 $\rightarrow$ 记录出库流水 $\rightarrow$ 生成实时库存报表**

## 3. 功能模块详细说明

### 3.1 采购计划管理（采购表）

这是整个系统的原始数据源 。该模块记录所有拟采购的物料详情，包括唯一的产品编号、项目归属、规格参数、预计数量及成本。其中，**产品编号**作为贯穿全系统的唯一标识符，是所有数据关联的核心。

### 3.2 物料主数据库（产品信息表）

该表充当系统的“标准字典”。它自动同步采购计划中的物料详情，将产品编号与对应的名称、规格型号、计量单位及分类进行绑定。其核心作用是确保在后续的入库和出库操作中，只需输入编号即可自动获取完整的物料描述，避免重复输入导致的人为错误。

### 3.3 入库流水记录（入库表格）

记录物料实际到货的情况。在录入产品编号后，系统会自动从主数据库中提取该物料的名称、规格和单位。操作人员仅需关注到货日期和实际入库数量。

### 3.4 出库流水记录（出库表格）

记录物料的领用或安装情况。与入库表逻辑一致，通过产品编号自动匹配物料详情，仅需录入出库日期和领用数量。

### 3.5 库存实时汇总（库存汇总）

这是系统的最终输出结果，提供物料的实时状态概览：

- **基本信息**：自动呈现物料的名称、规格、单位和类别。
- **入库总量**：自动汇总入库表格中该物料的所有到货数量。
- **出库总量**：自动汇总出库表格中该物料的所有领用数量。
- **期末库存**：通过计算“累计入库量”与“累计出库量”的差值，得出当前仓库中该物料的实际剩余数量。

## 4. 数据关联逻辑

系统采用了高度自动化的关联机制：

- **纵向同步**：采购表 $\rightarrow$ 产品信息表 $\rightarrow$ 入出库表。这意味着只要在采购计划中修改物料规格，全系统的相关描述将同步更新。
- **横向统计**：入出库流水 $\rightarrow$ 库存汇总。通过对产品编号的聚合统计，将零散的流水账单转化为结构化的库存报表。

## 5. 操作规范

- **先定义后使用**：所有物料必须首先在采购计划中定义编号并同步至信息表，方可在入出库表中进行记录。
- **唯一性原则**：严禁重复使用同一个产品编号给不同物料，以确保库存统计的准确性。
- **维护重点**：仅需维护采购表和入出库流水，汇总表和信息表由系统自动驱动，无需手动修改。
